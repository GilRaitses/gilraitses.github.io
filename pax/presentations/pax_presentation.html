A* Search with Unsupervised Cost Shaping from Pre-Trained Vision Models

Gil Raitses

Syracuse University

2025-12-08

Problem Statement

Traditional Pathfinding vs. Urban Reality

Problem: Traditional pathfinding minimizes distance, ignores perceptual stress

Traditional A* search (Hart, Nilsson and Raphael 1968) minimizes distance only.

Traditional A* Cost: \[ \text{cost}(n) = \text{distance}(n) \]

Stress-Aware Cost: \[ \text{cost}(n) = \text{distance}(n) + \alpha \times \text{stress}(n) \]

Challenge: How to compute perceptual stress from camera data?

Goal: Find paths that balance distance and stress (avoid crowded, prefer calm)

Key Points:

  • NYC has 967 traffic cameras citywide (Manhattan: 356 cameras)
  • Real-time stress varies by location, time, infrastructure
  • Human navigation considers more than just distance

Problem Space: 82 cameras in Midtown corridor (42nd-59th St), 554 intersections
Case Study: Grand Central → Carnegie Hall

System Overview

Stress-Aware A* Pathfinding

System Classification: Stress-aware A* pathfinding with unsupervised cost shaping from pre-trained vision models

Key Components:

  1. Perception (pre-trained vision models)
  2. Stress computation (unsupervised psychophysics)
  3. Cost shaping (basin/ridge detection)
  4. Pathfinding (deterministic A* search)

System Type: Deterministic A* search with hand-engineered costs (no reinforcement learning)

Methodology Overview

6-Step Pipeline

  1. Perception: Extract features from camera images (YOLO, CLIP, Detectron2)
  2. Stress Computation: Aggregate features into stress scores (unsupervised psychophysics)
  3. Spatial Interpolation: Map camera stress to intersections (street-based assignment)
  4. Cost Shaping: Detect ridges/basins and adjust stress (multi-scale analysis)
  5. Pathfinding: Find optimal paths using stress-weighted A* search
  6. Evaluation: Compare algorithms on real-world routes

Key Innovation: Unsupervised cost shaping from pre-trained vision models enables stress-aware pathfinding without labeled training data.

Perception Pipeline

Pre-Trained Vision Models

Models:

  • YOLO (Redmon et al. 2016): Object detection (pedestrians, vehicles) - COCO dataset
  • CLIP (Radford et al. 2021): Scene understanding (Vision Transformer via Hugging Face) - image-text pairs
  • Detectron2 (Wu et al. 2019): Instance segmentation (crowd density) - COCO dataset

Output: Camera-level features

  • Pedestrian counts, vehicle counts
  • Pedestrian spacing, infrastructure capacity
  • Semantic embeddings (512-dim)

Key Point: Pre-training applies ONLY to vision models, not pathfinding

Stress Computation (Unsupervised)

Psychophysics-Based Formula

Stress Formula: \[ \text{stress} = 0.35 \times \text{spacing stress} + 0.20 \times \text{infra stress} + 0.25 \times \frac{\text{pedestrians}}{50} + 0.20 \times \frac{\text{vehicles}}{20} \]

Spacing Stress (Power-Law): \[ \text{spacing stress} = \left(\frac{\text{personal space}}{\text{actual spacing}}\right)^{2.0} \]

Where: personal_space = 1.2 meters (proxemic zone) (Hall 1966)

Key Features:

  • Unsupervised (no labeled training data)
  • Based on psychophysics principles (Hall 1966)
  • Combines multiple perceptual factors

Next: Camera-level stress scores are interpolated to intersections using street-based assignment and Voronoi tessellation.

Spatial Interpolation & Cost Shaping

Street-Based Assignment

Spatial Interpolation:

\[ \text{stress}(\text{intersection}) = \text{stress}(\text{camera on same street}) \]

  • Enables block-by-block navigation
  • Falls back to nearest camera if no street-specific cameras

Voronoi Tessellation:

  • Camera stress scores interpolated to intersections using nearest-neighbor assignment
  • Street-based assignment ensures cameras on same street map to intersections on that street

Next: Multi-scale ridge and basin detection identifies high-stress barriers and low-stress channels to shape the cost function.

Ridge & Basin Detection (Multi-Scale)

Concept & Spatial Structure

Ridge Detection Score: \[ \text{ridge score} = 0.4 \times \text{local} + 0.15 \times \text{edge} + 0.25 \times \text{directional} + 0.2 \times \text{global} \]

If ridge_score > 0.75: corrected_stress = \max(\text{base stress}, 0.8)

Basin Detection Score: \[ \text{basin score} = 0.4 \times \text{local} + 0.15 \times \text{edge} + 0.25 \times \text{directional} + 0.2 \times \text{global} \]

If basin_score > 0.6: corrected_stress = \min(\text{base stress}, \text{target stress})

Pathfinding Algorithm

A* Search with Stress-Weighted Cost

Evaluation Function: \[ f(n) = g(n) + h(n) \]

Where: - \(g(n) = g(\text{prev}) + \text{distance} + \alpha \times \text{stress}(n)\) - \(h(n) = \text{Manhattan distance}(n, \text{goal})\) (admissible heuristic)

Path Cost: \[ \text{path cost}(\text{path}) = \sum[\text{distance} + \alpha \times \text{stress}] \]

Stress Weight: \(\alpha = 0.005\) (makes stress contribute ~50% of typical edge cost)

Key Features:

  • Deterministic search (not RL)
  • Admissible heuristic (guarantees optimality)
  • Stress-weighted edge costs

Diagnostic framing: A* expansion logic is held fixed across variants; objective weighting and heuristic condition are what change.

Frontier instrumentation: The frontier is instrumented to test behavior directly, measuring pruning effects in advance.

Next: The pathfinding algorithms are evaluated on a real-world route: Grand Central Terminal to Carnegie Hall.

Three Algorithm Variants

Stress Weight Clarification

Variant Role of α Typical Value
Weighted A* Weight on heuristic (ε in f=g+εh) 0.5
Stress-Aware A* Weight on stress in edge cost 0.005

Note: α plays two distinct roles: 0.5 is the standard Weighted A* factor on the heuristic; 0.005 is a tuned weight on stress in the edge cost.

A* Held Fixed; Heuristic/Weighting Varied

What was observed: Divergence appears at the trajectory level, with clear differences in frontier spread and explored-node volume across variants.

What was ruled out: In this OD pair, instrumentation did not show frontier collapse or admissible paths being pruned before expansion.

Algorithm Comparison Analysis

Why Algorithms Differ

  • Baseline A*: Distance-only objective, fewer explored nodes (57), shortest geometric route.
  • Weighted A* (α=0.5): Stress avoidance dominates objective weighting, yielding larger exploration (424) and longer detours.
  • Learned Heuristic: Intermediate explored volume (371) with a lower distance penalty than weighted A* in this test pair.

Frontier diagnostic reading: Differences are visible in search trajectory and expansion volume. For this OD pair, instrumentation does not indicate collapse or pre-expansion pruning of admissible candidates.

System Architecture (Detailed)

End-to-End Pipeline

  1. Perception: Pre-trained vision models extract features
  2. Stress Computation: Unsupervised formula converts features to stress
  3. Spatial Interpolation: Street-based assignment maps cameras to intersections
  4. Cost Shaping: Basin/ridge detection structures stress field
  5. Pathfinding: A* search minimizes path cost

Key Point: Hybrid perception-planning system with no learning in pathfinding stage

Limitations & Future Work

Phase 1 Limitations

Data Scope: - Single route evaluation: Grand Central → Carnegie Hall (1.7km) - Route chosen to test: (1) Avenue-level navigation (Park vs Madison), (2) Cross-street basin detection (54th Street), (3) Special infrastructure (6½ Avenue), (4) Typical Midtown pedestrian route - Limited camera coverage: 82 cameras (Midtown corridor, 42nd-59th St) - Image count: ~14,795 images (~180 per camera, Phase 1 bottleneck features) - Geographic coverage: ~0.5 square miles (3rd-9th/Amsterdam Ave) - Note: Manhattan-wide (356 cameras) and citywide (365 cameras) data already collected but not yet evaluated

Technical Limitations: 1. Connectivity gaps: 6½ Avenue, pedestrian passages require manual graph construction 2. Sensitivity to α: Stress-weight parameter requires manual tuning (α=0.005) - results vary with parameter choice 3. Static stress field: No temporal variance (rush hour vs off-peak, weekday vs weekend) 4. Single neighborhood: Midtown patterns may not generalize to residential areas

Validation Limitations: - No user study: Stress formula based on psychophysics principles, not validated against human perception - No ground truth comparison: Routes not compared to user preferences or other navigation systems

Graph & Algorithm: - Graph size: 554 intersections (limited to corridor) - No special infrastructure: Central Park paths, bridges not included - Deterministic only: No learning from user preferences or feedback
