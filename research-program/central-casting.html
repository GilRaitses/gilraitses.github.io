Central Casting

A recent, still-evolving method for keeping multi-repository agent work coherent.

The research in this gallery moves across several repositories. Over the last few months I started formalizing how that work stays coherent between sessions, so a new session can resume from recorded state. This describes the current approach at the level of structure. It is recent and still being tested, and parts of it are as much intention as settled habit.

Timeline, honestly

The pipelines and analysis tools came first, over several years. The coordination layer described below is new: the checkpoint method, the step logs, and the separation of roles were written in spring 2026. Before that, continuity across sessions was mostly ad hoc. So this is an early practice built on older work, young enough that its habits are still forming.

Checkpoints as memory

The current unit of memory is a checkpoint: a recorded state change in what a part of the work knows, can do, is blocked by, or is allowed to write. A checkpoint marks a real transition in state, at a coarser grain than a chat turn and a finer grain than task completion. A session reads the most recent checkpoints and resumes from there. The discipline is young, so it holds when it is followed and loosens when attention slips.

Agent homes as role contracts

Each part of the work has a defined home with explicit inputs, outputs, and an escalation rule, separating orchestration, execution, and research. Handoffs are written down so one step can take over from another without losing context. This structure is newer than the research it coordinates and is still settling.

A deliberate local and cloud boundary

The memory and audit layer stays local. Production work such as formatting, packaging, and cross-repository assembly can be delegated. Keeping that boundary explicit is a design choice: the connective tissue stays under local control, while heavier production steps can run elsewhere.

What is still hard

Context still drifts across long sessions, and a change in one repository reaches another only when it is carried there by hand. The checkpoint discipline reduces these failures while leaving some in place, and it leans on care more than on automation. Because the method is recent, the honest claim is modest: it helps, and it is still being proven.
