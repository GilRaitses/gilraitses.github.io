Continuity

An agent-run research practice, and how it runs through Cursor.

The research in this gallery moves across several repositories. Keeping that work coherent over weeks depends less on any single prompt and more on a method. This preview describes that method at the level of structure, not internal content.

Checkpoints as memory

The unit of memory is a checkpoint, a state change in what a lane knows, can do, is blocked by, or is allowed to write. It is not every chat turn and not only task completion. A session that starts today reads the most recent checkpoints rather than rereading the whole history, which is what lets long-running work continue without losing the thread.

Agent homes as role contracts

Each agent has a defined home with explicit inputs, outputs, and an escalation rule. Roles are separated into orchestration, execution, and research, and handoffs are written down. Because the contract is explicit, one agent can take over from another without context bleed.

A deliberate local and cloud boundary

The memory and audit layer stays local. Production work such as formatting, packaging, and cross-repository assembly can be delegated. Keeping that boundary explicit is a design choice: the connective tissue of the work is never handed off blindly, while the heavier production steps can run elsewhere.

What is still hard

Context still drifts across long sessions, and a change in one repository does not automatically propagate to another. The checkpoint discipline reduces these failures, but it depends on care rather than automation. Naming the limit is part of the practice.
